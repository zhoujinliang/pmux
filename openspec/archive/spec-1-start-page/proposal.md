# Proposal: 规格 1 - 默认启动页（选择工作区）

## 背景

pmux 是一个多 AI Agent 开发工作台，允许用户管理多个 git worktree 中的 agent 任务。为了让用户能够开始使用 pmux，需要一个清晰的入口点来选择他们想要管理的 git 仓库。

## 目标

实现规格 1：当用户首次启动 pmux 或没有保存的最近工作区时，展示一个友好的启动页面，引导用户选择一个 git 仓库作为工作区。

## 范围

### 包含的功能

1. **启动页展示**：应用启动时，如果没有已保存的最近工作区，则展示居中的启动页
2. **工作区选择**：提供明显的 CTA 按钮「选择工作区」，点击后打开系统文件夹选择器
3. **Git 仓库验证**：选择的目录必须是有效的 git 仓库（包含 `.git` 目录）
4. **工作区保存**：成功选择后，将工作区路径保存以便下次启动时恢复

### 不包含的功能

1. **tmux session 创建**：此规格不涉及 tmux 会话的创建或管理
2. **多 worktree 发现**：此规格不涉及发现和显示仓库内的 worktree
3. **终端渲染**：此规格不涉及任何终端视图的渲染
4. **状态检测**：此规格不涉及 agent 状态的检测或显示

## 用户体验流程

```
用户启动 pmux
  → 检查是否有保存的工作区
    ├─ 有 → 直接加载该工作区（规格 2+ 的流程）
    └─ 无 → 显示启动页
          → 用户点击「选择工作区」按钮
          → 系统文件选择器打开
          → 用户选择一个目录
          → 验证所选目录是否为 git 仓库
            ├─ 是 → 保存为工作区，进入规格 2 的流程
            └─ 否 → 显示错误提示，让用户重新选择
```

## 技术方案

### 架构

- 使用 `gpui-component` 构建 UI 组件
- 核心组件：`StartupPageComponent`、`AppRootComponent`
- 文件选择：使用系统原生文件选择器 API（`rfd` crate）
- 状态管理：简单的应用状态管理

### 组件设计

1. **AppRootComponent**：
   - 检查是否有保存的工作区
   - 根据状态决定渲染启动页或工作区内容

2. **StartupPageComponent**：
   - 居中布局，包含欢迎文案和 CTA 按钮
   - 响应「选择工作区」按钮点击事件

### 数据流

```
AppRootComponent
  → 检查本地存储中的工作区路径
  → 无路径 → 渲染 StartupPageComponent
  → 用户点击按钮 → 调用文件选择器
  → 选择路径 → 验证 Git 仓库
  → 验证通过 → 保存路径 → 触发工作区加载流程
```

## 成功标准

- 用户能够通过点击按钮打开文件选择器
- 选择有效 git 仓库后，应用能够识别并准备进入下一个流程
- 选择非 git 目录时，有适当的错误提示
- 选择的路径被正确保存，下次启动时不会再次显示启动页

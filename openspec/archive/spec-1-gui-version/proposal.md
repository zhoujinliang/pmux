# Proposal: 规格 1 GUI 版本 - 默认启动页（选择工作区）

## 背景

当前的规格 1 实现是 CLI（命令行）版本，虽然功能完整，但不符合 pmux 作为原生桌面应用的定位。用户期望看到图形界面而非文字交互。

## 目标

将规格 1 的 CLI 实现转换为完整的 GUI 版本，使用 `gpui-component` 构建原生桌面界面。

## 范围

### 包含的功能

1. **GUI 启动页**：使用 gpui-component 构建的居中启动页面
2. **现代化 UI**：美观的按钮、布局、颜色和字体
3. **系统文件选择器**：集成 rfd 的文件选择对话框
4. **实时反馈**：错误提示、加载状态等视觉反馈
5. **窗口管理**：适当的窗口标题、尺寸和位置

### 不包含的功能

1. **tmux 集成**（规格 2）
2. **终端渲染**（规格 2）
3. **Sidebar**（规格 2）

## 技术方案

- **UI 框架**: gpui-component
- **组件架构**: AppRoot → StartupPage / WorkspaceView
- **状态管理**: GPUI 的 Model/View 机制
- **文件选择**: rfd crate（已集成）

## 成功标准

- 应用启动后显示图形化的启动页
- 点击按钮打开系统文件选择器
- 选择有效 Git 仓库后显示成功状态
- 选择无效目录时显示图形化错误提示
- 窗口可以正常关闭和重新打开

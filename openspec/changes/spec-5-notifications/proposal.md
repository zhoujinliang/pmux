# Proposal: 规格 5 - 通知面板与错误提醒

## 背景

规格 4 已实现 Agent 状态检测，现在需要建立通知系统，及时告知用户重要事件（Error、Waiting 等），同时避免过度打扰。

## 目标

实现完整的通知系统：
1. 通知中心浮层展示历史通知
2. 智能通知触发（基于状态变化）
3. 静音/免打扰机制
4. 通知合并策略（防刷屏）

## 范围

### 包含的功能

1. **Notification 结构** - 通知数据模型
2. **NotificationCenter** - 通知管理与存储
3. **通知面板 UI** - 浮层展示组件
4. **智能触发器** - 基于状态变化自动生成通知
5. **静音系统** - 全局和按 pane 静音
6. **合并策略** - 30 秒内相同类型通知合并

### 不包含的功能

1. 系统原生通知（OS-level notifications）
2. 邮件/Slack 等外部通知渠道
3. 通知持久化到磁盘（仅内存存储）

## 用户体验流程

```
Agent 状态变化 (Error/Waiting)
  ↓
检查静音设置
  ├─ 静音中 → 忽略
  └─ 未静音 → 生成 Notification
            ↓
        检查合并窗口
          ├─ 同类型在 30s 内 → 合并计数
          └─ 新通知 → 添加到 NotificationCenter
                      ↓
                  TopBar 显示未读标记
                  可选：弹出 Toast
                      ↓
              用户点击铃铛图标
                      ↓
              打开通知面板浮层
                      ↓
              查看/标记已读/清除
```

## 技术方案

- **存储**: 内存中的 VecDeque（限制最大数量）
- **去重**: HashMap<(pane_id, type), Notification> 用于合并
- **UI**: gpui-component 浮层 + 列表
- **触发**: 监听 PaneStatusTracker 变更事件

## 成功标准

1. Error 状态立即生成通知
2. Waiting 状态生成通知
3. 30 秒内重复通知自动合并
4. 静音期间不生成通知
5. 通知面板正确显示历史
6. 标记已读后更新未读计数
